# dms-workspace — Epic Breakdown (2026-04-06)

## Overview

Two new epics derived from the 2026-04-06 epic descriptions file. This batch extends the project
backlog (Epics 1–48) starting at **Epic 49**. The work covers: fixing the Risk Group filter
dropdown width on the Universe screen so all options display on a single line, and switching the
dividend history data source from dividendhistory.org to dividendhistory.net (which correctly
finds securities that dividendhistory.org misses) while adding browser-like request headers for
rate limiting compliance.

All stories follow the existing project conventions:

- **Angular 21 zoneless** — `inject()`, `OnPush`, signal-first
- **SmartNgRX / SmartSignals** for state management and lazy loading
- **Vitest** for unit tests, **Playwright** for e2e
- **Tests are authoritative** — do not change tests, fix the code
- **`pnpm all`** must pass after every story
- **Playwright MCP server** must be used to verify UI fixes and create e2e tests where specified

  `apps/dms-material/src/app/global/global-universe/global-universe.component.html` (the
  `@case ('risk_group')` block). The Angular Material `panelWidth` input on `mat-select` (set to
  `"auto"`) is the canonical fix — the trigger stays column-width but the overlay panel expands to
  show the longest option on one line.
- The dividend history service is at
  `apps/server/src/app/routes/common/dividend-history.service.ts`. `BASE_URL` must change from
  `https://dividendhistory.org/payout` to `https://dividendhistory.net/payout`. Browser-like
  `fetch` request headers (User-Agent, Accept, Accept-Language, Referer) must be added to the
  `fetchAndParseHtml` helper. The existing 10-second rate limit should be retained (and documented
  as intentionally human-paced).
- The HTML parsing in `extractDividendJson` relies on a `<script data-dividend-chart-json>` tag
  — verify this pattern is also used on dividendhistory.net before committing the URL change.
- The log message `'falling back to Yahoo Finance'` in
  `get-consistent-distributions.function.ts` should be updated to remove the Yahoo Finance
  reference as part of this work.

## Epic 49: Fix Risk Group Filter Dropdown Width on Universe Screen

**Goal:** The Risk Group dropdown filter on the Universe screen must display all available options
on a single line in the open dropdown panel. Currently the panel is too narrow (constrained to the
column width), causing option labels to wrap and making the filter hard to use. The collapsed field
(the trigger before the user clicks) must remain the width of the column — only the open dropdown
panel needs to be wider.

---

### Story 49.1: Fix Risk Group Filter Dropdown Panel Width on Universe Screen

**Slug:** `49-1-fix-riskgroup-dropdown-panel-width`

As a trader,
I want the Risk Group filter dropdown panel on the Universe screen to be wide enough to show every
option on a single line,
so that I can easily read and select the correct risk group without text wrapping.

**Acceptance Criteria:**

**Given** a Universe screen column whose Risk Group filter dropdown contains options with long
names (e.g., "Tax-Free Income"),
**When** the user clicks the Risk Group filter trigger to open the dropdown,
**Then** all option labels are displayed on a single line with no wrapping inside the panel.

**Given** the Risk Group dropdown panel is open,
**When** the user reads the option list,
**Then** each option label is fully visible and untruncated within the dropdown panel.

**Given** the Risk Group filter is in its collapsed (closed) state,
**When** the column is rendered,
**Then** the trigger field width is still constrained to the column width (unchanged from before).

**Given** the fix is applied,
**When** `pnpm all` runs,
**Then** all unit tests pass with no regressions.

**Definition of Done:**

- [ ] `panelWidth="auto"` (or equivalent) added to the `mat-select` in the `@case ('risk_group')` block in `global-universe.component.html`
- [ ] Playwright MCP server confirms no option label wraps in the open dropdown panel
- [ ] Collapsed trigger field width is unchanged (still constrained to the column)
- [ ] `pnpm all` passes

## Epic 50: Fix Dividend History Load Failures (Switch to dividendhistory.net)

**Goal:** The `fetchDividendHistory` function currently fails to retrieve dividends for many
securities because dividendhistory.org does not list them (e.g., VFL, ACP). Manual testing
confirms dividendhistory.net returns data for these securities. The service must switch to
dividendhistory.net, add browser-like request headers to comply with the site's fair-use
expectations, and retain the existing rate limiting so that requests appear human-paced. After
the switch, VFL and ACP must successfully return dividend data.

---

### Story 50.1: Switch fetchDividendHistory to dividendhistory.net with Browser Headers

**Slug:** `50-1-switch-dividend-fetch-to-dividendhistory-net`

As a developer,
I want the `fetchDividendHistory` function to use dividendhistory.net instead of
dividendhistory.org and include browser-like request headers,
so that dividend data is retrievable for all securities, including VFL and ACP.

**Acceptance Criteria:**

**Given** the service is configured to use dividendhistory.net,
**When** `fetchDividendHistory('VFL')` is called,
**Then** a non-empty array of dividend rows is returned (not an empty array or null).

**Given** the service is configured to use dividendhistory.net,
**When** `fetchDividendHistory('ACP')` is called,
**Then** a non-empty array of dividend rows is returned.

**Given** the HTTP request to dividendhistory.net,
**When** the `fetch` call is made inside `fetchAndParseHtml`,
**Then** the request includes headers: `User-Agent` (a real browser UA string), `Accept`,
`Accept-Language`, and `Referer`.

**Given** the existing 10-second rate limiting,
**When** consecutive calls to `fetchDividendHistory` are made,
**Then** the delay between outbound HTTP requests is at least 10 seconds.

**Given** the HTML response from dividendhistory.net,
**When** `extractDividendJson` parses the response,
**Then** the `<script data-dividend-chart-json>` pattern successfully extracts the JSON data
(verify this pattern exists on dividendhistory.net before implementation).

**Given** the stale "falling back to Yahoo Finance" log message in
`get-consistent-distributions.function.ts`,
**When** any code path references that message,
**Then** the message is updated to no longer mention Yahoo Finance (e.g., "no dividend data
found for {symbol}").

**Given** all changes are applied,
**When** `pnpm all` runs,
**Then** all unit tests pass with no regressions.

**Definition of Done:**

- [ ] `BASE_URL` in `dividend-history.service.ts` changed to `https://dividendhistory.net/payout`
- [ ] Browser-like `fetch` request headers added to `fetchAndParseHtml`
- [ ] Manual or integration test confirms VFL returns dividend rows
- [ ] Manual or integration test confirms ACP returns dividend rows
- [ ] `extractDividendJson` verified to work with dividendhistory.net HTML structure
- [ ] Stale "falling back to Yahoo Finance" log message updated
- [ ] `pnpm all` passes
