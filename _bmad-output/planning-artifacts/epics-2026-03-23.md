# dms-workspace — Epic Breakdown (2026-03-23)

## Overview

This document provides epic and story breakdown for issues that remained unresolved or regressed after Epics 9–15. These are retry epics where previous implementation attempts were either incorrect in their diagnosis or incomplete in their fix. Starting at Epic 16.

---

## Requirements Inventory

### Functional Requirements

**FR16:** Storybook stories must render correctly using the `@nx/storybook` plugin, displaying actual components (not just building without errors). All stories must be verified via Playwright before being considered complete.

**FR17:** The wait/loading spinner must be centered both horizontally and vertically on the screen across all screens that use it (Screener, Universe, Account, Global Summary). The previous Epic 10 fix did not resolve the issue.

**FR18:** The Dividend Deposits table on the Account screen must display rows on initial load and maintain row visibility while scrolling. The previous Epic 12 fix resolved Open Positions and Sold Positions but Dividend Deposits remains broken due to its single-header-row structure.

**FR19:** The Global Summary screen must display the line graph to the right of the summary information and pie chart on desktop screens (≥1024px) and stack vertically on smaller screens. On cold start, the graph must not overlap the pie chart. The previous Epic 13 fix did not resolve the issue.

**FR20:** The Account Summary screen must display the line graph to the right of the summary information on desktop screens (≥1024px) and stack vertically on smaller screens. The previous Epic 14 fix did not resolve the issue.

### Non-Functional Requirements

_No new NFRs — these are retry bug fixes where prior implementations were either diagnostically incorrect or incomplete._

### Additional Requirements

- **Context from Epics 9–14:** Each of these epics had a previous implementation attempt that either did not fully resolve the issue or was incorrectly diagnosed as resolved.
- **Storybook plugin:** The user has specified that `@nx/storybook` (the official NX plugin) must be used. The previous implementation did not use this plugin. Reference: https://nx.dev/docs/technologies/test-tools/storybook/introduction
- **Pre-Epic 8 reference:** For layout and spinner issues, the git history from just before Epic 8 (the Tailwind CSS migration) should be consulted to understand what was working before.
- **Playwright verification (MANDATORY):** All visual fixes and story rendering MUST be verified using the Playwright MCP server. A fix is not complete until Playwright confirms the component or layout renders correctly. Building without errors is insufficient.
- **Dividend Deposits single-header difference:** The Dividend Deposits table differs from Open Positions and Sold Positions in that it has only one header row. This structural difference is likely the cause of its continued breakage.

## Epic 18: Fix Dividend Deposits Table (Retry)

**Goal:** Fix the Dividend Deposits table on the Account screen so that rows are displayed correctly on initial load and remain visible while scrolling. The previous Epic 12 fix resolved Open Positions and Sold Positions tables but did not fully fix Dividend Deposits.

**Context:** The Dividend Deposits table has a key structural difference from Open Positions and Sold Positions: it has only **one header row**, while the other two tables have **two header rows**. This difference likely causes the table's virtual scroll or layout container to calculate dimensions incorrectly. The symptoms are:

1. No rows are displayed on initial page load (only the scrollbar is visible)
2. After scrolling to the bottom and back, rows appear
3. Scrolling down again causes the rows to disappear again

This scroll-triggered visibility pattern strongly suggests a virtual scroll or container height calculation issue that is sensitive to the number of header rows.

**Dependencies:** Requires Epic 8 (Tailwind migration) to be complete. Builds on Epic 12. Supersedes the Dividend Deposits portion of Epic 12.
