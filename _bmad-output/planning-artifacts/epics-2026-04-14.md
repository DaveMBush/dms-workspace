### Additional Requirements (Architecture / Context)

- **Residual distributions bug — Epic 73:** The Epic 71 fix updated `updateExistingUniverseRecord`
  in `apps/server/src/app/routes/universe/sync-from-screener/index.ts` to write
  `distributions_per_year` from the `getDistributions` response. If the 5 specified symbols still
  show 1, the root cause has shifted: the bug is now either inside `getDistributions` itself
  (returning 1 for these specific ticker symbols) or in how CefConnect encodes distribution
  frequency for closed-end fund structures. The symbols OXLC, NHS, DHY, CIK, and DMB are all
  closed-end funds — check whether CefConnect's CEF-specific response fields differ from the ETF/
  stock frequency fields currently being mapped in
  `apps/server/src/app/services/distributions/get-distributions.function.ts`. dividendhistory.net
  confirms 12 distributions per year for all five. Story 73.1 must use the Playwright MCP server
  to trigger a live sync and then query the database for these symbols' `distributions_per_year`
  values to confirm which are still incorrect before inspecting code.

- **CSV import residual error — Epic 74:** Unlike the Epic 69 regression (which was a 400 before
  any processing), the current error occurs mid-import — "enough got done" implies some rows were
  inserted before the error. This is consistent with a failed constraint (e.g. a duplicate key,
  a missing FK, or a field-length violation), an unhandled transaction type code in the CSV parser,
  or a data-type conversion failure on a specific field. The import pipeline lives in:
  `apps/server/src/app/routes/import/index.ts` → `fidelity-import-service.function.ts` →
  `fidelity-csv-parser.function.ts`. Story 74.1 must identify the exact error (log line, function,
  failing row type) using the Playwright MCP server to upload a synthetic fixture CSV that
  reproduces the error, without touching production code. Synthetic fixture files should cover:
  Buy, Sell, Cash Dividend, Stock Split — the same types present in a real Fidelity export.

- **System integration E2E test — Epic 75:** The user's manual test procedure is:
  (1) Assert that tables `screener`, `universe`, `trades`, `divDeposits` are empty before starting.
  (2) Navigate to `http://localhost:4201/global/screener`, click "Refresh", wait for overlay.
  (3) Navigate to `http://localhost:4201/global/universe`, click "Update Universe", wait for overlay.
  (4) Verify `distributions_per_year` for OXLC, NHS, DHY, CIK, DMB via the table UI or a direct

  (e.g. `DELETE /api/test/reset`) or a direct Prisma call from a Node fixture, not via UI.
  Fixture CSV files must be stored at `apps/dms-material-e2e/fixtures/` and must not contain real

## Epic 73: Residual Monthly-Payer Distribution History

**Goal:** Even after the Epic 71 fix that updated `updateExistingUniverseRecord` to write
`distributions_per_year` from the `getDistributions` return value, five specific monthly-paying
closed-end fund symbols — OXLC, NHS, DHY, CIK, and DMB — continue to show
`distributions_per_year = 1` after a screener-to-universe sync. The root cause is no longer in
the write path; it is either within `getDistributions` itself returning 1 for these specific
CUSIPs, or in a mismatch between how CefConnect encodes distribution frequency for closed-end
fund structures versus the field mapping currently implemented. This epic investigates the exact
root cause, fixes it, and verifies with a live sync that all five symbols receive `12`.
