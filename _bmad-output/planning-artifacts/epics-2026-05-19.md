# dms-workspace — Epic Breakdown (2026-05-19)

## Overview

Three new epics derived from the 2026-05-19 epic descriptions file. This batch extends the project backlog from **Epic 106** through **Epic 108**. All three are bug fixes:

- **Epic 106 — Janky Scrolling After Account/Filter Change (Round 9):** The scrolling fixes in Epics 29, 31, 44, 60, 64, 87, 101, and most recently 105 (Round 8) have eliminated scrolling artifacts on freshly loaded screens. Artifacts return when changing the active account or applying/clearing a filter without a full page reload. This epic investigates the state reset issue triggered by context changes and applies a targeted fix with regression coverage for the account-change / filter-change scenario.

- **Epic 107 — Close Position Should Immediately Remove from Open Positions List:** On the Open Positions screen, when a position is closed by setting `Mst Rcnt Sll Dt` to a date and `Mst Rcnt Sell $` to a positive value, the row should disappear immediately. The previous attempt in Epic 104 has not resolved the issue — the row still only disappears after a page refresh. This epic re-investigates the broken reactivity layer (save mutation, store update, selector reaction, or missing refetch) and applies a fresh fix with regression coverage. A suggested approach from the bug report is to use `handleSocketNotification('top', 'update', ['1']);` when the saved row transitions to closed, telling `top` it needs to reload itself and children.

- **Epic 108 — Fix Linux Installation and AppImage Migration Failure:** When attempting to install the app on Linux via AppImage, the installation completes but running the app fails with a migration error: "Could not update the database schema. Error: Migration failed: Invalid params: missing field `migrationsList`. The application will now exit." This epic investigates Prisma migration deployment in the Electron AppImage context and applies a fix for cross-platform installation with E2E verification.

### Additional Requirements (Architecture / Context)

- **Scrolling Round 9 search space (Epic 106):** Epic 105 (Round 8) and Epic 101 (Round 7) and earlier rounds fixed scrolling for freshly loaded screens. The regression returns specifically when the active account or a filter changes *without a full reload*. Candidates: (a) CDK `cdk-virtual-scroll-viewport`'s internal data-source / range-renderer is not reset when the underlying signal swaps out; (b) sticky header's containing block is re-created (or not re-created) on account change; (c) SmartNgRX/SmartSignals selector returns a new array reference with stale row identities; (d) an ancestor `transform`/`will-change`/`contain` is conditionally applied during account-change loading state; (e) Story 105.2's fix addressed one trigger but not the other. Investigation (Story 106.1) must test each hypothesis and review what Story 105.2 actually changed.

- **Open Positions list reactivity (Epic 107):** Open Positions rows are sourced from the Trade DTO. The open vs closed filter is either server-side (the Open Positions endpoint excludes closed trades) or client-side (the SmartNgRX/SmartSignals selector filters by sell-date / sell-price). The bug indicates that after save, the client view doesn't re-evaluate the open/closed predicate. Investigation (Story 107.1) must identify the broken layer: (a) save mutation doesn't update store entity; (b) store entity updates but selector is not reacting; (c) open/closed split is server-side and screen isn't refetching; (d) Epic 104's fix did not actually ship or did not address the right layer. The bug report suggests `handleSocketNotification('top', 'update', ['1']);` as a possible fix path — Story 107.2 should evaluate whether this is the right surface for the fix.

- **Prisma migration in Electron (Epic 108):** The error "missing field `migrationsList`" suggests that Prisma's migration execution in the Electron context is not finding or correctly initializing the migrations list. Investigation must distinguish between: (a) the AppImage bundle missing the `prisma/migrations` folder; (b) Prisma not finding the schema file in the packaged context; (c) the migration runner being invoked with incorrect arguments in the Electron environment; (d) insufficient file permissions in the AppImage sandbox; (e) a Prisma version mismatch between dev and packaged builds.
