### Additional Requirements (Architecture / Context)

- **Shared SQLite test database:** The E2E suite uses a shared SQLite database at
  `apps/dms-material-e2e/test-database.db`. Because `workers: 1` and `fullyParallel: false` are
  already configured in `playwright.config.ts`, tests run serially — but a test that leaves
  database rows behind (e.g. a partially-committed import, an orphaned trade, a leftover screener
  row) will pollute the state seen by later tests. The most common isolation failure pattern is a
  test that relies on an empty table (or a specific row count) receiving leftover data from a
  previous test that did not clean up after itself. Any `beforeEach` / `afterEach` / `afterAll`
  cleanup in the spec files should be audited in Story 76.1.

- **Chromium vs. Firefox divergence:** Five tests fail under Chromium and six under Firefox. The
  one test that fails in Firefox but not Chromium is worth examining first: it is most likely a
  timing difference (Firefox is generally slower to settle on network requests and DOM mutations
  in Playwright), a browser-specific rendering difference, or a `locator` strategy that resolves
  differently across browsers. The Playwright MCP server can be pointed at each browser in turn
  to reproduce the divergent failure.

- **Retries already configured:** `retries: 2` (local) and `retries: 3` (CI) are set in
  `playwright.config.ts`. A test that is consistently failing (not flaky) will exhaust all retries
  and still fail. If a test passes on one retry but not another, it is flaky due to timing and
  requires a more robust wait strategy rather than a code change. Story 76.1 must distinguish
  between consistent failures and flaky failures.

- **Known pre-existing Epic 42 context:** Epic 42 (`Fix Failing E2E Tests`) is recorded in
  `sprint-status.yaml` with both stories at `ready-for-dev`. The current 2026-04-17 failures may
  overlap with or be distinct from the Epic 42 backlog items. Story 76.1 must cross-reference the
  Epic 42 story files (if they exist) to avoid duplicating work. If the current failures are
  entirely covered by Epic 42's scope, Story 76.1 should document this and Epic 76 can be
  superseded by Epic 42. If the failures are new or distinct, Epic 76 proceeds independently.

- **E2E test location and commands:**
  - Test files: `apps/dms-material-e2e/src/*.spec.ts`
  - Chromium run: `pnpm e2e:dms-material:chromium`
  - Firefox run: `pnpm e2e:dms-material:firefox`
  - Individual test isolation run: `pnpm nx run dms-material-e2e:e2e --project=chromium --grep "<test name>"`
  - Config: `apps/dms-material-e2e/playwright.config.ts`

## Epic 76: Fix Failing E2E Tests (Chromium + Firefox)

**Goal:** Five E2E tests fail when the full suite is run under Chromium; six fail under Firefox.
Some or all of these tests may pass when run in isolation, which would indicate test isolation
problems (shared database state not cleaned up between tests, localStorage or cookie state leaking
across specs, or timing assumptions broken by a prior test's side effects). Others may fail even
in isolation, indicating genuine functional regressions. This epic follows a strict
diagnose-first sequence: Story 76.1 identifies exactly which tests fail, whether they fail in
isolation or only in the full suite, and documents the root cause for each. Story 76.2 fixes any
isolation issues found. Story 76.3 fixes any functional regressions found. At the end of the
epic, `pnpm e2e:dms-material:chromium` and `pnpm e2e:dms-material:firefox` both exit with
status 0.

isolation or only when run as part of the full suite — so that Story 76.2 and Story 76.3 can
