## Epic 34: Find a Better Dividend Source

**Goal:** Replace Yahoo Finance dividend fetching (which truncates to 3 decimal places) with a
higher-precision free source. The candidate sources are `dividendhistory.org` and
`dividendhistory.net`. One source must be evaluated, selected, implemented as a server service, and
wired into the existing dividend update process as a replacement for Yahoo Finance dividend data.

**Scope:** Server-side only. The Angular frontend is not affected. The existing Yahoo Finance
*price* fetching (used in "Update Fields") is out of scope — only the dividend/distribution data
fetch is replaced.

---

**Slug:** `34-1-evaluate-and-select-dividend-data-source`

As a developer,
I want to evaluate dividendhistory.org and dividendhistory.net (and any other viable free sources)
against the dividend data needs of the DMS application,
so that I can select the best source before writing any integration code.

**Acceptance Criteria:**

**Given** the requirement to fetch dividend amounts with ≥ 4 decimal-place precision,
**When** dividendhistory.org is evaluated (manually or via automated HTTP request),
**Then** the evaluation determines whether it provides dividend history for standard US CEF tickers
with ≥ 4 decimal places and without a paid API key requirement.

**Given** dividendhistory.net is also evaluated,
**When** the comparison is complete,
**Then** a short decision record (`_bmad-output/implementation-artifacts/dividend-source-evaluation.md`)

demonstrating the actual data shape returned.

**Definition of Done:**
- [ ] `dividend-source-evaluation.md` created with all required sections
