# dms-workspace — Epic Breakdown (2026-05-30)

## Overview

Three new epics derived from the 2026-05-30 epic descriptions file. This batch extends the backlog from **Epic 114** through **Epic 116**. The work covers disconnected-header table layout behavior, modal symbol lookup independence from Universe filtering, and unnecessary repeat loading on Account Summary screens.

- **Epic 114 — Fix Disconnected-Header Table Layout and Scrollbar Behavior:** Restore the intended disconnected-header table behavior so headers remain fixed, the vertical scrollbar stays pinned at the far right of the table area, and header/body alignment remains correct.
- **Epic 115 — Decouple Modal Symbol Lookup from Universe Filters:** Make symbol lookup in the Add Position and Dividend/Deposit modals independent from filtered Universe state by moving to an unfiltered, query-driven symbol source.
- **Epic 116 — Stop Continuous Reloading on Account Summary Screens:** Eliminate the summary-screen reload loop so data loads once on open and only refreshes for legitimate state changes.

All stories follow existing project conventions:

- **Angular 21 zoneless** — `inject()`, `OnPush`, signal-first
- **SmartNgRX / SmartSignals** for state management
- **Fastify** backend, **Prisma** ORM (SQLite dev / PostgreSQL prod)
- **Vitest** for unit tests, **Playwright** for E2E (Chromium + Firefox)
- **Tests are authoritative** — do not change tests unless a test is asserting something we are explicitly changing
- **`pnpm all`** must pass after every story
- **Playwright MCP server** must be used to reproduce and verify all UI failures
