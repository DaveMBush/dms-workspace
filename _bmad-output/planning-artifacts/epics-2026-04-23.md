# dms-workspace — Epic Breakdown (2026-04-23)

## Overview

Five new epics derived from the 2026-04-23 epic descriptions file. This batch extends the project
backlog starting at **Epic 83** through **Epic 87**. The work covers:

- **Epic 83 — Electron App Documentation and Build Fix:** The Electron shell created in Epic 77
  needs two things: documentation explaining how to configure the `BrowserWindow` to load the
  built Angular app and handle Angular Router navigation, and a fix for the broken build process.
  Stories cover auditing what is broken, writing the documentation, and fixing the build.

- **Epic 84 — Volatility Visible for All Symbols and New Icons:** The "Vol" column added in
  Epic 81 only shows volatility icons for symbols that have positions, which is a data-filtering
  bug that must be fixed so all universe symbols display volatility regardless of position status.
  Three additional icon categories are also needed — flat, up-then-down, and down-then-up — to
  give a fuller picture of distribution behaviour over time.

- **Epic 85 — Store Volatility in Universe Table:** Volatility is currently calculated on the
  fly at query time. This is unnecessary work on every page load since volatility only changes
  when distribution data changes. This epic pre-calculates volatility (both long-term and
  short-term) and stores it in the `universe` table, wiring recalculation triggers to all data
  change events so the stored values stay current.

- **Epic 86 — Short Term Volatility (SVol) Column:** Following Epics 81 and 85, add a "SVol"
  column as the 2nd column on the Universe screen (immediately after "Vol") showing the 1-year
  distribution volatility. The column reuses the same icon set as "Vol" but is driven by the
  stored short-term volatility value introduced in Epic 85.

- **Epic 87 — Janky Scrolling Returns (Round 6):** The scrolling regression has recurred. This
  time it affects not only the Universe screen but also Open Positions, Sold Positions, and
  Dividend Deposits. Previous attempts (Epics 29, 31, 44, 60, 64) each fixed the issue
  temporarily. This epic uses the Playwright MCP server to identify and document the current
  failure mode across all affected screens, implements a permanent fix, and builds a
  comprehensive regression prevention suite to stop the cycle.

All stories follow existing project conventions:

- **Angular 21 zoneless** — `inject()`, `OnPush`, signal-first
- **SmartNgRX / SmartSignals** for state management
- **Fastify** backend, **Prisma** ORM (SQLite dev / PostgreSQL prod)
- **Vitest** for unit tests, **Playwright** for E2E
- **Tests are authoritative** — do not change tests unless a test is asserting something we are
  explicitly changing
- **`pnpm all`** must pass after every story
- **Playwright MCP server** must be used to reproduce and verify all UI failures
