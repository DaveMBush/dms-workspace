# dms-workspace — Epic Breakdown (2026-05-27)

## Overview

Two new epics derived from the 2026-05-27 epic descriptions file. This batch extends the project backlog from **Epic 112** through **Epic 113**. Both address regressions or instabilities introduced in or exposed by recent work on the Universe screen and the base table component.

- **Epic 112 — Fix Post-Refactor Layout Regressions in Two-Region Base Table:** The Epic 111 two-region refactor eliminated sticky-header scroll jank but introduced three layout regressions: (1) the vertical scrollbar drifts with the table content on narrow viewports instead of staying pinned to the screen's right edge, (2) the scrollbar sits immediately to the right of the table content on wide viewports rather than at the far right of the available area, and (3) columns no longer expand to fill available space and the empty area beyond the table content uses a mismatched background color. This epic diagnoses all three regressions, applies the fixes, and adds regression coverage.

- **Epic 113 — Fix Universe Sticky Sort State Loss:** Under undetermined circumstances, the Universe screen's sticky sort state resets and Dave is forced to re-sort manually. The sort state should persist across all normal interactions. This epic identifies the exact conditions that trigger the loss and applies a durable fix with test coverage.

- **SmartNgRX / SmartSignals** for state management
- **Fastify** backend, **Prisma** ORM (SQLite dev / PostgreSQL prod)
- **Vitest** for unit tests, **Playwright** for E2E (Chromium + Firefox)
- **Tests are authoritative** — do not change tests unless a test is asserting something we are explicitly changing
- **`pnpm all`** must pass after every story

### Additional Requirements (Architecture / Context)

- **Vertical scrollbar positioning (Epic 112, R1 & R2):** The root cause of both scrollbar issues is almost certainly that `overflow-y: auto` is applied to an inner element that is sized to the table content width rather than to a full-viewport-width outer container. The fix should place vertical scrolling (`overflow-y: auto`) on an outer wrapper that spans the full available width so the scrollbar always appears at the right edge of that wrapper. The inner table content region handles horizontal scrolling (`overflow-x: auto`). Use the Playwright MCP server to verify the scrollbar position before and after the fix.

- **Column expand-to-fill and background color (Epic 112, R3 & R4):** Before Epic 111, the base table used CSS auto-layout (or `flex: 1` on columns) to distribute spare space. After Epic 111 the column widths are explicit and fixed, so the table is sized to content and spare space is exposed as an unstyled region. The fix should let columns stretch to fill available space — for example by setting `min-width` per column and `width: 100%` / `table-layout: fixed` on the table itself, or by distributing spare space via flexbox — so the table always fills the container. The beyond-table empty area, if any remains, should inherit or be set to the same background as the cell background (typically a CSS variable from the Angular Material theme). Investigate the current column model and SCSS in `base-table.component.ts` / `.scss` before deciding on the approach.

- **Universe sort persistence (Epic 113):** The Universe screen already has a sticky-sort mechanism (introduced in Epic 24/25). Story 113.1 must determine under exactly what interaction(s) the sort state is cleared — look for signal resets, NgRx state reinitialisation, SmartSignals context invalidation, or `contextId` changes that might flush the saved sort state. The investigation story produces Dev Notes; no production code is changed.

## Epic 112: Fix Post-Refactor Layout Regressions in Two-Region Base Table

**Goal:** The Epic 111 two-region refactor eliminated sticky-header scroll jank but introduced three observable layout regressions. On narrow viewports the vertical scrollbar moves with the table content instead of staying fixed at the screen's right edge. On wide viewports the scrollbar appears next to the table content rather than at the far right of the available space. Before Epic 111 the columns spread to fill the available width; now they are content-sized, and the uncovered area to the right uses a visually jarring background color. This epic fixes all three regressions in the base table component and locks in regression coverage so they do not recur.

---
