# dms-workspace — Epic Breakdown (2026-04-20)

## Overview

Six new epics derived from the 2026-04-20 epic descriptions file. This batch extends the project
backlog starting at **Epic 77** through **Epic 82**. The work covers:

- **Epic 77 — Electron Shell:** Wrap the Angular client and Fastify backend in an Electron shell
  so the application can be distributed and run as a desktop app. Internal navigation must remain
  in-app; external links must open the system browser; API calls must be handled by the embedded
  Fastify process.

- **Epic 78 — Dist/Year Weekly Validation Fix:** The Dist/Year field on the symbol edit screen
  rejects the value 52 with "Value must be at most 12", incorrectly constraining weekly-frequency
  symbols. The UI validation maximum must be raised to 52 when the symbol's frequency is weekly.

- **Epic 79 — OXLC Joint Brokerage Missing Positions:** OXLC positions for the Joint Brokerage
  account are absent from the application despite being present in the Fidelity CSV. The CSV
  import and stock-split processing logic must be audited, corrected, and re-validated to ensure
  all rows survive the pipeline intact and appear in the UI.

- **Epic 80 — Error Logs Screen Restore:** A prior refactor (Epic 70 / Epic 1) accidentally
  routed the Error Logs menu item to a stub component instead of the original file-viewing
  component. The original component — which lets Dave view and delete individual error log files —
  must be recovered from git history and restored.

- **Epic 81 — Distribution Volatility Indicator:** Add a "Vol" column as the first column on the
  Universe screen that displays a per-symbol distribution stability icon (steady / increasing /
  decreasing / volatile) calculated from the last 1 year and last 5 years of distribution data.

- **Epic 82 — UnSkip Tests:** Every skipped unit test and E2E test in the codebase must be
  triaged: duplicates are deleted; tests covering unique behaviour are unskipped and fixed until
  `pnpm all` passes with zero skipped tests.

All stories follow existing project conventions:

- **Angular 21 zoneless** — `inject()`, `OnPush`, signal-first
- **SmartNgRX / SmartSignals** for state management
- **Fastify** backend, **Prisma** ORM (SQLite dev / PostgreSQL prod)
- **Vitest** for unit tests, **Playwright** for E2E
- **Tests are authoritative** — do not change tests unless a test is asserting something we are
  explicitly changing
- **`pnpm all`** must pass after every story
- **Playwright MCP server** must be used to reproduce and verify all UI failures
