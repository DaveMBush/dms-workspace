### Additional Requirements (Architecture / Context)

- **Distribution frequency (Epic 58):** `calculateDistributionsPerYear` in
  `apps/server/src/app/routes/settings/common/get-distributions.function.ts` uses only the last 2
  past rows; if fewer than 2 past rows are returned by `fetchDividendHistory` the function defaults
  to 1 (annual). The bug is likely that dividendhistory.net returns mostly future ex-dates with only
  one or zero past rows for a new symbol, leaving the algorithm without enough data to infer
  frequency. Story 58.1 must confirm the exact data shape returned before any fix is written.

- **Dark mode filename colour (Epic 59):** The `.selected-file-name` class in
  `apps/dms-material/src/app/global/import-dialog/import-dialog.component.scss` uses
  `color: var(--mat-sys-on-surface-variant, rgba(0, 0, 0, 0.6))`. The fallback is always near-black,
  making the text invisible against the dark dialog background. Additionally,
  `apps/dms-material/src/styles.scss` hardcodes `.mat-mdc-dialog-surface { background-color:
  #ffffff !important }` which forces the dialog background to white even in dark mode — this must
  also be corrected. Both files should use Material 3 theme tokens that respond to the active theme.

- **Janky scrolling (Epic 60):** The same scrolling regression (jank / stutter / blank rows during
  fast scroll) has been fixed and re-broken across Epics 29, 31, and 44. Story 60.1 must use the
  Playwright MCP server to reproduce the behaviour, identify the latest regression path, and author a
  failing E2E test before any code changes are made. The fix documentation should be thorough enough
  to explain the architectural constraint that causes this to keep recurring.

- **CUSIP-as-symbol reverse split (Epic 61):** The pre-split "YOU BOUGHT" rows in the provided CSV
  use CUSIP `691543102` as the symbol field (not the ticker symbol `OXLC`). OXLC is the *known
  failing example* of a general class of bug: any symbol whose lots were imported under a raw CUSIP
  will not be adjusted when a reverse split arrives with the ticker as the symbol. The reverse-split
  "FROM" row uses `OXLC` as the symbol with CUSIP `691543102` embedded in the action field; the
  "TO" row uses `691543102` as the symbol. Story 61.1 uses OXLC as the concrete test case to
  reproduce the bug; the fix implemented in Story 61.2 must be **symbol-agnostic** — it must
  resolve any CUSIP to its ticker before querying for lots, not just the OXLC CUSIP.

## Epic 58: New Symbol Distribution Cycle

**Goal:** When a symbol is added to the Universe (either through CSV import or the + button),
`getDistributions()` is called and stores `distributions_per_year` in the database. The stored
value is currently defaulting to 1 (annual) because `calculateDistributionsPerYear` requires at
least 2 past distribution rows to infer the cadence, and dividendhistory.net frequently returns
only future rows (or a single past row) for newly-added symbols. This epic diagnoses the exact data
shape returned for a newly-added symbol and then ensures the frequency algorithm has enough data to
correctly set `distributions_per_year` for annual / quarterly / monthly / weekly payers.

## Epic 61: Stock Splits — CUSIP-Symbol Lot Adjustment

under a raw CUSIP symbol (rather than a ticker), the lot-adjustment step fails to find and update
them. **OXLC is the known failing example** — the four OXLC pre-split lots (300, 150, 500, 580
shares at prices $4.50, $4.49, $4.06, $3.44) are not being adjusted to the correct post-split

import pipeline must resolve any CUSIP to its ticker before querying for open lots to adjust,
ensuring correct behaviour for all symbols — not just OXLC. This epic uses the OXLC CSV data as
the concrete test case to drive the fix.
