## Epic 51: Correct Risk Group Filter Dropdown Panel Width (`panelWidth=""`)

**Goal:** Epic 49 applied `panelWidth="auto"` to the Risk Group `mat-select` on the Universe
screen. The Angular Material documentation states that `panelWidth="auto"` sets the panel overlay
to match the trigger width — which is the same as the column width and does not help. The correct
value is `panelWidth=""` (empty string), which removes any width constraint on the panel so it
grows to display the widest option on a single line. This epic replaces the incorrect value with
the correct one and updates the e2e tests to match the new behaviour.

---

### Story 51.1: Change Risk Group Dropdown `panelWidth` from `"auto"` to `""` and Verify with Playwright

**Slug:** `51-1-fix-riskgroup-panelwidth-empty-string`

As a trader,

on a single line,

**When** the developer changes the value to `panelWidth=""`,

**Given** the updated template is served,

## Epic 53: Properly Evaluate and Fix dividendhistory.net Integration

**Goal:** Epic 50 changed `BASE_URL` in `dividend-history.service.ts` to
`https://dividendhistory.net/payout` and added browser-like request headers, but the agent
never actually inspected the live site to confirm how it works. The `<script data-dividend-chart-json>`
HTML pattern assumed in Story 50.1 may not match what dividendhistory.net actually returns. This
epic mandates that the site is investigated first using the Playwright MCP server for the specific
test symbols VFL, ACP, IFN, and FAX — and that the implementation is then fixed based on what the
site actually returns, not what was assumed.
