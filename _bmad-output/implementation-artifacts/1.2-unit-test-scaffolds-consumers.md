# Story 1.2: Unit test scaffolds for base-table consumer components (skip)

Status: Approved

## Story

As a developer,
I want to create/verify unit test scaffolds and assertions for all consumer components BEFORE any code changes,
So that tests exist before implementation and can be un-skipped once the implementation is complete.

## Acceptance Criteria

1. [Unit test scaffolds written for all 5 consumer component spec files]
2. [All NEW assertions use `.skip` so they are not executed yet, but the scaffold compiles]
3. [The scaffolds cover: binding to base-table inputs/outputs, cellTemplate rendering, filterRowTemplate rendering, template stability]
4. [Each consumer's unique behavior is covered: editable cells, delete action, symbol search filter, viewChild(BaseTableComponent) reference]
5. [This story is documentation/scaffolding only — NO functional code changes to the components or templates]
