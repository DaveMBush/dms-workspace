# Story 1.5: Update base-table E2E regression tests for new `<mat-table>` DOM

Status: Approved

## Story

As a developer,
I want to update the base-table E2E regression tests to work with the new `<mat-table>` DOM structure,
So that automated table layout testing continues to catch regressions with the new implementation.

## Acceptance Criteria

1. [The base-table E2E test file `base-table-two-region-regression.spec.ts` has all selectors updated from DIV-based classes to mat-table equivalents]
2. [All tests still verify the same behaviors: vertical header scroll, body scroll, header-body scroll sync, virtual scroll rendering]
3. [Tests pass in chromium AND firefox]
4. [No regressions in the base-table or consumer E2E tests]

## Tasks / Subtasks

- [ ] Step 1: Read the current `base-table-two-region-regression.spec.ts` completely and identify ALL DIV-based selectors to replace:
  - `.dms-table-shell` → `<mat-table>` or `[role="table"]` mat-table element
  - `.dms-table-header` → `.mat-table-header` or `<mat-header-row>` element
  - `.dms-header-row` → mat-header-row element
  - `.dms-filter-row` → filter row element (may not apply with new structure)
  - `.dms-column-header-row` → mat-header-row element + columns
  - `.dms-header-cell` → `mat-header-cell` element within each column
  - `.dms-table-body` → `cdk-virtual-scroll-viewport` or `.mat-table-body`
  - `.dms-body-row` → `mat-row` element
  - `.dms-body-cell` → `mat-cell` element
  - `.dms-outer-scroller` → outer scroller wrapper (if still needed)
  - `.dms-table-header-viewport` → header scroll container
- [ ] Step 2: Update each locator to use mat-table-compatible selectors:
  - `[role="rowgroup"]` for header/body rowgroups
  - `mat-header-cell` for header cells
  - `mat-cell` for body cells with column index
  - `mat-row` for rows
  - `mat-column-{field}` CSS classes for column targeting
  - Filter row: look for elements with input/checkbox inside mat-header-row
- [ ] Step 3: Update scroll synchronization tests — verify the scroll sync between header and body still works by:
  - Scroll the body horizontally
  - Verify the header's `scrollLeft` mirrors the body
  - Use `page.evaluate(() => viewport.elementRef.nativeElement.scrollLeft)`
- [ ] Step 4: Update virtual scroll tests — verify:
  - `cdk-virtual-scroll-viewport` renders the correct number of items
  - `cdkVirtualFor` tracks each row
  - Rows outside the viewport are not rendered (virtualization works)
- [ ] Step 5: Update checkbox/selection tests — change from `[class.dms-checkbox]` to `mat-checkbox` selectors
- [ ] Step 6: Run `pnpm e2e:dms-material:chromium` — pass
- [ ] Step 7: Run `pnpm e2e:dms-material:firefox` — pass
- [ ] Step 8: Run all other consumer E2E tests to verify no regressions

## Dev Notes

### E2E Test Files Affected

1. **Base-table E2E (MAIN TARGET)**: `apps/dms-material-e2e/src/base-table-two-region-regression.spec.ts`
   - This is the primary file to update. It tests:
     - The two-region layout (header viewport + body scroll)
     - Horizontal scroll sync between header and body
     - CDK virtual scroll behavior
     - Selection checkboxes
     - Column rendering
   
2. **Dividend Precision (MAYBE affected)**: `apps/dms-material-e2e/src/dividend-precision.spec.ts`
   - Uses `await expect(page.locator('dms-base-table')).toBeVisible()`
   - This should still work (targets the host component, not internal DOM)
   - Verify after updating the main regression test

3. **Account Sort/Filter Persistence (MAYBE affected)**: `apps/dms-material-e2e/src/account-sort-filter-persistence*.spec.ts`
   - Uses selectors targeting inside the consumer components, which internally use base-table
   - Selectors may need updating if they target `.dms-*` classes
   - Update if tests fail

### Selector Migration Reference

| Old Selector (DIV-based) | New Selector (mat-table) |
|--------------------------|--------------------------|
| `.dms-table-shell` | `[role="table"] mat-table` |
| `.dms-table-header` | `mat-header-row` or `.mat-table-header` |
| `.dms-header-row` | `mat-header-row` |
| `.dms-filter-row` | First `mat-header-row` inside each `mat-table` (or filter row container) |
| `.dms-column-header-row` | `mat-header-row` with `mat-column-` cells |
| `.dms-header-cell` | `mat-header-cell` or `[mat-header-cell]` |
| `.dms-table-body` | `cdk-virtual-scroll-viewport` or `.cdk-virtual-scrollable-content` |
| `.dms-body-row` | `mat-row` or `cdk-column-virtual-scroll-item` |
| `.dms-body-cell` | `mat-cell` or `[mat-cell]` |
| `.dms-checkbox` | `mat-checkbox` |
| `.dms-outer-scroller` | The scroller wrapper element (may not exist with new structure — verify) |

### Anti-Patterns to Avoid

- **DON'T** change the test logic — only the selectors/DOM queries
- **DON'T** update consumer E2E tests (that's Story 1.6's job)
- **DON'T** remove any test assertions — only update the selectors
- **DON'T** modify non-E2E test files
- **DON'T** assume the two-region scroll pattern still exists — it may have changed
