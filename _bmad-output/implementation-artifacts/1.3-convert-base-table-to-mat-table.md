# Story 1.3: Convert base-table component to `<mat-table>` with ngContainer matColumnDef

Status: Approved

## Story

As a developer,
I want to rewrite the `base-table` component template and related code to use `<mat-table [dataSource]="dataSource">` with `<ng-container matColumnDef>` columns,
So that the table layout follows Angular Material's recommended pattern, avoids sticky-header complications, and no longer needs the DIV-based two-region layout with header/table separation.

## Acceptance Criteria

1. [The component template uses `<mat-table [dataSource]="dataSource">` directly — no `<table>` wrapper element]
2. [Columns are defined with `<ng-container matColumnDef="{{col.field}}">` for each column, including a "select" column when `selectable=true`]
3. [The template uses `<mat-header-cell *matHeaderCellDef>` (with `col.header`), `<mat-cell *matCellDef="let row">` (with row data and cellTemplate), `<mat-header-row *matHeaderRowDef="displayedColumns">`, and `<mat-row *matRowDef="let row; columns: displayedColumns">` inside `<mat-table>`]
4. [The previously skipped unit tests from Story 1.1 are now un-skipped (`.skip` removed) and the existing unit test scaffolds pass]
5. [The component still exposes the EXACT SAME public API (9 inputs, 4 outputs, ContentChildren, ViewChildRefs, 13 public methods) with unchanged signatures]
6. [CDK virtual scrolling still works for large datasets (`cdk-virtual-scroll-viewport` wraps the content)]
7. [Horizontal scroll sync between header and body still works]
8. [Selection (single and multi with select-all) still works via CDK SelectionModel]
9. [Sorting via `onSort`/`onHeaderClick`/`getAriaSort` still works with sort rank indicators and ARIA sort attributes]
10. [The component compiles without TypeScript errors (`pnpm build` succeeds)]
11. [CellType built-in formatting still works (`currency`, `date`, `number` types trigger pipes)]

## Dev Notes

### Architecture Compliance — What MUST be preserved

The `base-table` component is the shared table primitive for the entire application. Breaking its API would cascade into all 5 consumer components.

**Preserved contract (do NOT change):**
- `<dms-base-table>` selector
- All 9 inputs: `data` (req), `columns` (req), `tableLabel`, `rowHeight`, `bufferSize`, `selectable`, `multiSelect`, `loading`, `sortColumns`, `contextId` (same types and defaults)
- All 4 outputs: `sortChange`, `rowClick`, `selectionChange`, `renderedRangeChange`
- Both ContentChildren: `@ContentChild('cellTemplate')`, `@ContentChild('filterRowTemplate')`
- All 3 ViewChildRefs: `viewport`, `headerScrollViewport`, `outerScroller`
- All 13 public methods with same signatures
- `ColumnDef` interface (`column-def.interface.ts`) — shape must NOT change

### Key Implementation Details

1. **No `<table>` wrapper** — Per the Epic input, use `<mat-table>` directly. This is the key to avoiding sticky-header problems.

2. **CDK Virtual Scroll** — Keep `cdk-virtual-scroll-viewport` but structure it to work with `<mat-table>`. The viewport wraps each cell's content with `*cdkVirtualFor`.

3. **Column widths** — `ColumnDef.width` is `number`. Use `[style.width.px]="col.width || 100"` on `<mat-header-cell>` and `<mat-cell>`.

4. **Filter row** — The `filterRowTemplate` renders filter inputs in the header. Render it as an extra row above the column headers within `<mat-table>`.

5. **Selection column** — Create `<ng-container matColumnDef="select">` only when `selectable()` is true. It contains `<mat-checkbox>` in the header (for select-all) and `<mat-checkbox>` in each body row. Prepend `'select'` to `displayedColumns`.

6. **CDK Virtual Scroll inside mat-cell** — The virtual scroll viewport wraps the body content. Each cell's inner content (text, pipes, cellTemplate) is rendered inside the cdkVirtualItem.

7. **Header scroll sync** — Previously: `bindHeaderInteractions()` listened to body scroll and mirrored to header. Now: listen to `cdk-virtual-scroll-viewport`'s inner scroller `scrollLeft`.

### ColumnDef Interface (DO NOT CHANGE)

```
interface ColumnDef {
  field: string;
  header: string;
  tooltip?: string;
  width?: number;     // width in pixels, applied via style.width.px
  sortable?: boolean;
  editable?: boolean;
  type?: 'actions' | 'boolean' | 'currency' | 'custom' | 'date' | 'number' | 'text';
}
```

- **DON'T** change the `ColumnDef` interface shape
- **DON'T** change the public API contract
- **DON'T** remove CDK virtual scroll
- **DON'T** break `cellTemplate` or `filterRowTemplate` extensibility
- **DON'T** change consumer templates/components
- **DON'T** remove the `.skip` from consumer tests (Story 1.2) — that's Story 1.4's job
- **DON'T** update E2E tests yet (that's Story 1.5/1.6's job)
