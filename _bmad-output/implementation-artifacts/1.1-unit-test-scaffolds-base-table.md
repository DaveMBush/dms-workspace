## Dev Notes

### What to Test Coverage

**Public API (preserved through migration):**

Inputs:
- `data` (input.required<T[]>): row data
- `columns` (input.required<ColumnDef[]>): column definitions
- `tableLabel` (input<string>): ARIA label default 'Data table'
- `rowHeight` (input<number>): row height for CDK, default 57
- `bufferSize` (input<number>): virtual scroll buffer factor, default 10
- `selectable` (input<boolean>): show check boxes, default false
- `multiSelect` (input<boolean>): select all checkbox, default false
- `loading` (input<boolean>): progress bar, default false
- `sortColumns` (input<SortColumn[]>): initial sort specs, default []
- `contextId` (input<string|null>): scroll-to-top trigger, default null

Outputs:
- `sortChange` (output<Sort>): emitted on sort click
- `rowClick` (output<T>): emitted on row click
- `selectionChange` (output<T[]>): emitted on selection toggles
- `renderedRangeChange` (output<ListRange>): emitted on scroll (debounced 100ms)

ContentChildren:
- `cellTemplate`: custom cell rendering, receives `$implicit`, `column`, `index`
- `filterRowTemplate`: custom filter row, receives `$implicit` = ColumnDef

ViewChildRefs:
- `viewport` = CdkVirtualScrollViewport
- `headerScrollViewport` = ElementRef<HTMLElement>
- `outerScroller` = ElementRef<HTMLElement>

Public Methods (13):
1. `isExpired$(row)` → boolean
2. `getIsCef$(row)` → true|null
3. `getHasPosition$(row)` → true|null
4. `gainLossType$(row)` → 'gain'|'loss'|'neutral'|undefined
5. `gainLossClassMap$(row)` → Record<string,boolean>
6. `getLastShiftKey()` → boolean
7. `onSort(sort)` → void
8. `onHeaderClick(column)` → void
9. `getAriaSort(column)` → string|null
10. `onRowClick(row)` → void
11. `toggleSelection(row)` → void
12. `isAllSelected()` → boolean
13. `toggleAllRows()` → void
14. `trackByFn(index, item)` → string
15. `scrollToTop()` → void

Computed:
- `dataSource`: sorted copy of data
- `displayedColumns`: column fields + optional 'select'
- `filterColumns`: column fields + 'Filter' suffix
- `sortRankMap`: maps sort active col to rank string
- `selection`: SelectionModel<T>

### Anti-Patterns to Avoid

- **DON'T** change component implementation — this is test scaffolding only
- **DON'T** remove existing tests — append to them
- **DON'T** un-skip the assertions yet — wait for Story 1.3
- **DON'T** modify the ColumnDef interface (it's in a separate file)
