## Dev Notes

### What Consumers Change (and What They Don't)

Consumer **templates** should NOT need modification because the public API contract is preserved. Consumers bind:
- `[data]`, `[columns]`, `[sortColumns]`, `[contextId]`, `[bufferSize]`, `[tableLabel]`, `[loading]`, `[selectable]`, `[multiSelect]`, `[rowHeight]`
- `(sortChange)`, `(rowClick)`, `(selectionChange)`, `(renderedRangeChange)`
- `<ng-template #cellTemplate>`, `<ng-template #filterRowTemplate>`

Consumer **SCSS** uses `dms-base-table { display: block; height: 100%; width: 100%; }` — this targets the host component element which hasn't changed (still `<dms-base-table>`).

Consumer **tests** from Story 1.2 have `.skip` on new assertions. This story un-skips them.

Consumer **files to modify** (if needed):
1. `apps/dms-material/src/app/account-panel/dividend-deposits/dividend-deposits.component.spec.ts` — un-skip
2. `apps/dms-material/src/app/account-panel/open-positions/open-positions.component.spec.ts` — un-skip
3. `apps/dms-material/src/app/account-panel/sold-positions/sold-positions.component.spec.ts` — un-skip
4. `apps/dms-material/src/app/global/global-universe/global-universe.component.spec.ts` — un-skip

### What This Story Un-skips

This story removes `.skip` from the unit test scaffolds written in Story 1.2. After the implementation is verified correct, the dev agent should remove `.skip` and run tests before marking complete.

### Anti-Patterns to Avoid

- **DON'T** change consumer templates — the contract is preserved; if a template breaks, the base-table's API has a bug
- **DON'T** change the ColumnDef interface
- **DON'T** remove the ColumnDef import from consumer files
- **DON'T** update E2E tests (that's Story 1.5/1.6) — consumer tests are unit tests (Story 1.2 scaffolds → un-skipped here)
- **DON'T** change consumer component logic or services
